# DocSync Agent

Агентный сервис для автоматического обновления документации на основе изменений в коде.

## Какая задача решается

В большинстве проектов документация быстро устаревает. Разработчики вносят изменения в код (новые параметры функций, поведение, команды и т.д.), но документация обновляется редко или с задержкой.

Это приводит к нескольким проблемам:

- пользователи получают неверную информацию
- онбординг новых разработчиков усложняется
- документация теряет доверие
- ревью PR требует дополнительного времени для проверки документации

## Для кого этот сервис

- команды разработки, поддерживающие open-source проекты
- команды с API или библиотеками
- проекты с документацией в GitHub (README, docs/, MkDocs)

## Какая боль сейчас

Сейчас разработчики должны вручную:

1. понять, какие части документации затронуты изменениями
2. найти соответствующие страницы документации
3. обновить описание функций, параметров или примеров

На практике до этого часто не доходят.

## Что делает DocSync Agent

DocSync Agent автоматически анализирует пулл реквесты и:

1. изучает изменения в коде
2. определяет, какие части документации могут быть затронуты
3. генерирует предложения по обновлению документации
4. добавляет изменения документации прямо в Pull Request

В случае неясностей агент может задать уточняющие вопросы разработчику через Telegram.

## Что именно делает PoC на демо

На демо PoC система сможет:

1. реагировать на новый PR в GitHub
2. анализировать изменения
3. находить потенциально затронутые страницы документации
4. генерировать обновление документации
5. добавлять изменения документации в PR

Пример сценария:

- разработчик добавляет новый параметр функции
- агент обнаруживает изменение
- агент обновляет документацию функции
- изменения добавляются в тот же PR

## Что не делает PoC (out of scope)

Чтобы ограничить сложность проекта, PoC не будет:

- как-либо менять код проекта помимо документации
- поддерживать все языки программирования
- выполнять глубокий анализ кода
- автоматически переписывать большие разделы документации
- выполнять сложную интерпретацию бизнес-логики

Также PoC будет работать только:

- с GitHub
- с документацией в Markdown в MkDocs
- с ограниченным количеством файлов документации
