# Спецификация модуля Serving / Config

## Цель

Описать минимальную эксплуатационную конфигурацию PoC.

## Режим запуска

PoC допускает два режима:

- webhook режим для реального GitHub события;
- ручная проверка для локального тестирования на сохранённом PR snapshot.

## Конфигурация

Обязательные параметры:

- `GITHUB_WEBHOOK_SECRET`
- `GITHUB_TOKEN`
- `LLM_PROVIDER`
- `LLM_MODEL`
- `LLM_TIMEOUT_SEC`
- `DOC_PATH_ALLOWLIST`
- `MAX_DIFF_LINES`
- `MAX_DOC_CANDIDATES`
- `TELEGRAM_BOT_TOKEN`

## Версии моделей

Для PoC:

- одну основную модель для генерации;
- при необходимости более дешёвую модель для классификации/реранкинга.

Изменение версии модели должно логироваться как изменение в конфигурации, чтобы можно было сравнивать качество.

## Секреты

- секреты только через env или менеджер секретов;
- секреты запрещено писать в репозиторий и в обычные логи.

## Релизы

- конфигурация должна поддерживать dry-run;
- режим публикации переключается флагом: `comment_only` или `commit_patch`;
- все лимиты должны быть конфигурируемыми без изменений кода.
