# Спецификация модуля Tools / APIs

## Цель

Инкапсулировать все внешние side effects и сделать их управляемыми, наблюдаемыми и безопасными.

## Поддерживаемые интеграции

### GitHub API

Операции:

- принять webhook;
- получить PR metadata;
- получить список измененных файлов и diff;
- прочитать содержимое doc-файлов;
- создать коммент;
- создать коммит или обновить ветку с патчем на доки.

Контракт ошибок:

- `auth_error`
- `rate_limited`
- `not_found`
- `conflict`
- `transient_http_error`

Timeout:

- короткие read requests: до 10 секунд;
- publish requests: до 15 секунд;
- retry: 2-3 попытки с exponential backoff только для transient ошибок.

Side effects:

- comment в PR;
- commit в ветку PR.

Защита:

- публикация только после validator;
- проверка, что меняются только разрешённые файлы;
- идемпотентность по `session_id` и `head_sha`.

### LLM API

Операции:

- классификация сценария;
- генерация структурированного патча;
- формирование вопроса для разработчика.

Контракт:

- вход всегда содержит system policy и контекст;
- выход всегда должен соответствовать схеме.

Ошибки:

- `timeout`
- `invalid_schema`
- `provider_error`
- `context_overflow`

Timeout:

- до 20 секунд на один вызов;
- не более 2-4 вызовов на PR в идеальном сценарии.

Защита:

- строгая схема на ответ;
- температура низкая;
- секреты и токены не передаются в prompt.

### Telegram API

Операции:

- отправить один уточняющий вопрос;
- получить ответ в рамках активной сессии.

Ошибки:

- `delivery_error`
- `timeout_waiting_reply`

Timeout:

- ожидание ответа асинхронное, TTL 24 часа для PoC.

Защита:

- только текстовые вопросы без чувствительных данных;
- один активный вопрос на один PR;
- отсутствие ответа не блокирует систему.
