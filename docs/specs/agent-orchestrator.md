# Спецификация модуля Agent / Orchestrator

## Шаги

1. `ingest`
2. `load_pr`
3. `analyze_diff`
4. `retrieve_docs`
5. `build_context`
6. `generate`
7. `build_patch`
8. `validate`
9. `publish`
10. `clarify`
11. `complete`

## Правила переходов

- `analyze_diff -> retrieve_docs` только для поддерживаемых типов изменений;
- `retrieve_docs -> generate` только если уверенность retrieval выше порога;
- `generate -> build_patch` только при валидной схеме;
- `validate -> publish` только при полном прохождении проверок;
- `validate -> clarify` при низкой уверенности;
- `publish -> complete` при успешном принятии.

## Условия остановки

- патч опубликован;
- коммент опубликован;
- отправлен вопрос человеку и сессия переведена в ожидание;
- превышен бюджет на ретраи;
- PR не подходит под реализуемые типы изменений.

## Политика ретраев

- GitHub/LLM ошибки: ограниченный ретрай;
- schema error от LLM: максимум один повтор с промптом на устранение ошибок;
- повторяющиеся ошибки: переход в коммент или `failed`.
