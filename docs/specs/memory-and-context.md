# Спецификация модуля Memory / Context

## Цель

Давать модели только рабочий контекст текущего PR и не накапливать лишний контекст.

## Session state

Основная сущность:

- `PRSession`

Поля:

- `session_id`
- `repo`
- `pr_number`
- `head_sha`
- `stage`
- `change_intent`
- `retrieval_result`
- `llm_decision`
- `validation_report`
- `human_clarification`
- `outcome`

## Memory policy

- память short-term и привязана к PR;
- после завершения PR сохраняются только артефакты для будущего анализа;
- полные трейсы и только для дебага;
- личные данные пользователя не используются как источник памяти.

## Context assembly

Контекст LLM состоит из:

- system policy;
- краткой карточки PR;
- diff summary;
- 1-3 наиболее релевантных отрывков документации;
- чёткой инструкции по выходной схеме.

Не включаются:

- весь diff целиком без фильтрации;
- полные большие Markdown-файлы;
- обсуждения PR;
- история предыдущих LLM вызовов.

## Бюджет

- лимит на размер промпта;
- если бюджет превышен, сначала режется объём diff, затем отрывки документации;

## Логирование

- артефакты сессий: ограниченный TTL;
- metrics/logs: дольше, без полного содержимого;
- clarification replies: только до закрытия задачи или истечения TTL.
